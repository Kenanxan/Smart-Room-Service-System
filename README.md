# Smart Room Service System

This repository is a public showcase extracted from a private Nush.ai project.

Its purpose is simple:

- show that the product exists
- communicate the product direction and UX surface
- share a few representative frontend samples without exposing private code

## Important Scope Note

The real production project remains private.

This public repository is intentionally limited to a sanitized frontend showcase with mock data. It does not contain the complete application.

## What Is Intentionally Excluded

- backend source code
- database schema and migrations
- authentication and authorization flows
- environment files and secrets
- deployment and infrastructure configuration
- live API routes, socket wiring, and internal service logic
- customer, hotel, supplier, courier, or operational data
- sensitive business rules and private implementation details

## What This Repo Contains

- a static preview page that explains the system at a high level
- representative frontend sample files derived from the private product direction
- mock data only
- documentation describing the public-safe scope

## High-Level Technology Stack Of The Private Product

- Frontend: React, React Router, Material UI, Tailwind CSS
- UI and localization tooling: Chart.js, i18next, Formik, Leaflet
- Backend: Node.js, Express, Socket.IO
- Data and storage: PostgreSQL, Knex, AWS S3
- Authentication and security: JWT-based authentication and server-side security middleware

## Why The Repo Is Structured This Way

The private project includes production integrations and internal workflows that should not be published in a public repository. This version is intentionally reduced so viewers can understand the product concept without exposing anything sensitive.

## Security Boundary

Before publishing, the public copy was separated from the original project and stripped of:

- backend directories
- `.env` files
- real base URLs
- auth tokens
- socket configuration
- export tools
- operational scripts

Please keep this repository sample-only. If new files are added later, they should remain mock-based and public-safe.
